    Собирает проект с конфигурацией отладки
buildAllRelease
    Собирает проект с конфигурацией релиза

configDev
    Устанавливает конфигурацию игрового сервера для разработки
configMap
    Устанавливает конфигурацию игрового сервера для создания карт
configServer
    Устанавливает конфигурацию игрового сервера в соответствии с реальным сервером для игры

runQuickAll
    Запускает клиент и сервер без сборки
runQuickClient
    Запускает клиент без сборки
runQuickServer
    Запускает сервер без сборки

Небольшое пояснение по поводу вариаций файлов.

Релизная сборка содержит различные оптимизации, в то время как отладочная сборка содержит инструменты отладки. Если вы составляете карту, используйте сборку release (buildAllRelease), так как она будет работать более гладко.

Конфигурационный файл Server просто соответствует реальному серверу, в то время как два других имеют некоторые настройки cvar, которые могут пригодиться полезными для конкретных задач. (dev - отладка, map - создание карт)
По сути, это только экономит вам несколько команд при загрузке, но все равно очень удобно.

Обязательно иметь при запуске сервера:
1)Git или один из многих сторонних пользовательских интерфейсов, которые облегчают его использование. Убедитесь, что он установлен в PATH.
2)Python 3.7 или выше. Обязательно установите его в PATH в Windows. Вы должны получить python с сайта python.org. Версии, установленные из магазина windows, иногда вызывают проблемы со сборкой.
3).NET 7.0 SDK или выше. Visual Studio также устанавливает его, если вы работаете под Windows.

Опционально:
ARM (M1) Пользователи Mac: Вам нужно убедиться, что вы установили x64 .NET, а не ARM .NET. В настоящее время движок не работает на Mac ARM, поэтому рекомендуется использовать x64 через эмуляцию Rosetta 2.
Желательно наличие IDE, чтобы разработка не была мучительной (все бесплатные варианты, если не указано иное):
Для ++Windows++, Visual Studio 2022 Community. Для минимальной установки вам понадобится рабочая нагрузка для разработки .NET desktop, компилятор C#, поддержка C#, менеджер пакетов NuGet, MSBuild и .NET 7 SDK или что-то в этом роде.
Для ++macOS++, Visual Studio для Mac.
Для ++всех платформ++, (НЕ БЕСПЛАТНО) Rider - одна из лучших доступных IDE, и многие разработчики SS14 предпочитают ее Visual Studio. Студенты колледжей/университетов могут получить бесплатную образовательную лицензию, даже если они не изучают информатику.
Для ++всех платформ++, Visual Studio Code с расширением C#. Обычно уступает по возможностям полноценным IDE, таким как обычная Visual Studio, но некоторым опытным программистам нравится минимализм.
